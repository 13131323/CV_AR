# 연구 제목 및 최종 목표

연구 제목 : 실시간 장면 이해 기반 환경 적응형 AR 아바타 행동 생성 연구

- 연구 제목 후보
    
    단안 RGB 영상 기반 실시간 지형-의미론적 추론과 LLM을 결합한 AR 아바타의 자율적 행동 생성 파이프라인 연구
    

연구 질문 : 단안 RGB 영상 기반 장면 이해와 LLM 행동 판단을 결합하면, AR 아바타가 현실 공간에서 물리적으로 자연스러운 행동을 자율적으로 생성할 수 있는가?

최종 목표 : 스마트폰 카메라로 촬영한 실내 공간을 인공지능이 이해하고, LLM이 상황에 맞는 행동을 판단하며, 3D 아바타가 현실 지형에 맞게 물리적으로 어색함 없이 이동하고 행동하는 통합 파이프라인 구축

- 장면 이해 결과를 아바타 행동으로 변환
    - 장면을 보고 → 스스로 판단하고 → 자연스러운 동작을 생성하는 아바타
- ‼️ 실내 공간을 촬영된 사진으로 할 것인가 실시간 동영상으로 할 것인가?
    - 촬영된 사진 한 장 : 정적인 이미지 안에 아바타 생성
        - 너무 간단한 구현이 될 수 있음
        - AR이라기보다는 이미지 분석에 가까워짐
    - 촬영된 사진 여러 장 : 다각도 사진 여러 장을 촬영하여 3D 공간 생성
        - 3D 공간 생성이 필요함 (구현 난이도 증가)
    - 실시간 동영상 : 포켓몬 고와 같은 AR 서비스
        - 구현 난이도가 높음
        - 매 프레임마다 비전 모델을 돌려야 하므로 연산량이 엄청남
        - 카메라가 흔들릴 때마다 3D 공간 정보가 튀어 아바타가 순간이동하는 문제가 있음

실시간 동영상으로 하기로 결정

# 문제 정의

기존 연구의 한계 : 기존의 가상 아바타나 에이전트는 미리 정의된 가상 환경(Virtual Scence)에서만 움직이거나, 현실 공간을 단순한 장애물 지도로만 인식하여 사물의 용도(Affordance)에 맞는 복잡한 상호작용이 불가능했음

- ‼️ 기존의 AR이 미리 정의된 가상 환경에서만 움직이거나 사물의 용도에 맞는 복잡한 상호작용이 불가능했다는 객관적인 근거가 필요함
- 기존 Vision 기반 장면 이해 결과는 객체 수준의 정보에 머물러 있어 행동 가능한 표현으로 직접 활용하기 어렵다. 또한 LLM은 환경의 물리적 제약과 행동 가능성을 충분히 반영하지 못해 비현실적인 행동을 생성하는 문제가 있다.
    - ‼️ 객관적인 증거 필요
- 만약 이것이 사실이고 연구 및 개발에 성공했다면 무인 제조 현장의 AI 도입에 활용 가능

“2. 관련 연구 조사 및 정리”에서 확인할 것

해결하고자 하는 문제 : 현실의 시각 데이터(Vision)와 언어적 판단(LLM)을 결합하여, 아바타가 현실 공간의 물리적·기능적 특성을 모두 이해하고 자연스럽게 행동하게 만드는 것

# 연구의 범위 (In-Scope vs Out-of-Scope)

연구에 포함할 내용

- 공간 범위
    - 실내 정적 환경
    - 일단 실내 정적 환경으로 하지만 파인튜닝에 따라 여러 공간에 적용 가능하도록 하는 확장성이 필요
- 사물 범위
    - 일상 도구
    - 일단 몇몇 일상 도구로 하지만 파인튜닝에 따라 여러 사물에 적용 가능하도록 하는 확장성이 필요
- 아바타의 행동
    - 정의한 사물에 맞는 동작
- 입력 데이터
    - 현실 세상 사진

연구에서 제외할 내용

- 새로운 비전/LLM 모델의 밑바닥부터 학습 (Scratch traing)
- 실시간 물리적 시뮬레이션의 완벽한 구현 : 아바타의 미세한 구현이 아닌 물리적 접촉(IK)에만 집중

연구 고려 항목

- 동적 사물과의 상호작용 : 움직이는 사람이나 동물에 대한 인지 및 소통
- 실시간 처리 최적화 vs 정확도 개선

# 핵심 기여도 정의 (Expected Contribution)

기술적 통합 : 비전 인지, LLM 판단, 그래픽스 모션을 하나의 Embodied AI 시스템으로 엮은 End-to-End 파이프라인 제안

- AR 환경에서 ‘3D 아바타가 현실 지형의 물리적 제약(바닥 높낮이, 장애물 등)을 고려해 사람처럼 걷고 행동을 생성하는 지각-행동 결합(Sense-Act) 파이프라인
- Scene understanding 결과를 affordance-aware action space로 변환하는 방법 제안
- Scene → affordance → action으로 이어지는 표현 구조 제안

현실성 개선 : 단순한 경로 이동을 넘어 어포던스(Affordance)를 활용해 현실 공간의 사물과 자연스럽게 상호작용하는 아바타 구현

- 환경 제약을 반영한 action candidate 생성 구조 설계
- LLM이 물리적 가능성을 고려하도록 하는 입력 표현 방식 제안

확장성 : AR/XR 환경에서 현실 공간 데이터를 기반으로 가상 캐릭터가 공존할 수 있는 메타버스/디지털 트윈의 기초 기술 제공

- 산업 현장에 AI를 도입할 때 실제 현장을 보고 판단하는 에이전트 기술

# 고려사항

공간 입력 방식

- 단일 이미지 → AR이 아닌 정적 이미지
- 이미지 여러 장 → AR이라기보다는 실시간 3D 디지털 공간 생성
- 실시간 동영상 → AR 그 자체

실시간 동영상으로 하는 수밖에 없음

- 사용자가 버튼을 누르거나 앱이 켜진 직후 첫 번째 프레임을 캡쳐해서 이를 바탕으로 상호작용
    - 초기 포켓몬 고 방식으로 추정
    - 서버의 비전 AI가 사물(의자, 테이블)의 3D 좌표를 딱 한 번만 추출해서 스마트폰으로 돌려줌
        - 혹은 AR로 좌표 추출
    - 비전 AI가 알려준 의자의 좌표를 유니티의 AR Foundation(ARCore/ARKit) 공간 안에 심어둠
    - 아바타는 서버가 처음에 정해준 현실 의자 위치에 실시간으로 완벽하게 고정된 채 상호작용
- 초기 이미지에서 이탈하는 경우의 문제
    - 지형의 고정 (Spatial Anchoring) : 비전 AI가 추출한 사물(의자, 테이블)의 3D 좌표는 자이로스코프 기반의 공중 좌표가 아니라, AR Foundation이 감지한 '바닥면(y=0)' 위에 단단히 고정(Snapping)
    - 카메라 시점의 실시간 추적 (Visual-Inertial Odometry) : AR Foundation은 스마트폰의 센서뿐만 아니라 카메라로 들어오는 실시간 동영상에서 주변의 특징점들을 계속 추적
- 너무 어려우면 이 문제를 Out-of-Scope로 처리

아바타와 상호작용하는 사물의 움직임

- 방법 1. 오브젝트 위치로 이동 → 집는 애니메이션 → 끝
    - 애니메이션을 1회 재생하고 끝
    - 연구 주제에 크게 벗어나지 않음 (파이프라인이 그대로 있음)
    - 시각적인 어색함이 있음
- 방법 2. “가짜 오브젝트”를 만들어서 붙인다
    - 유니티 프리팹 사용
    - 현실 오브젝트와 가짜 오브젝트의 완벽한 정합성이 필요함
- 방법 3. 비전 인페인팅
    - 아바타가 오브젝트와 접촉하는 순간 원래 오버젝트를 텍스처로 덮어씌우는 방식
    - 너무 어려움..

일단 방법 1로 하다가 여유가 있을 것 같으면 방법 2로 하자

아바타 생성 크기

- 수치 기반 자동 스내핑 사용 (Grounding)
    - 현실 세계의 바닥을 기준으로 삼음
    - 아바타의 키를 고정 (1.7m 정도)
    - 사물 높이에 맞춰 아바타의 크기 최종 보정

# 기타 사항

파이프라인 구조

- 장면 이해 → 행동 결정 → 행동 표현 생성 → 동작 생성

범위 설정

- 인지 : 공간 파악, 사물 용도 파악
- 판단 : LLM 기반 행동 결정
- 행동 : NavMesh 경로 계획 및 물리적 동작 생성 (IK)

시맨틱 : 의미

어포던스 : 행동 가능성

Embodied AI : 인공 지능에서 인터페이스 에이전트라고도 하는 구현된 에이전트는 해당 환경 내의 물리적 신체를 통해 환경과 상호 작용하는 지능형 에이전트입니다.

System Paper (Integration Paper) : 일반적으로 서로 다른 이론, 데이터, 연구 결과, 또는 관점들을 하나로 결합하고 종합하여 새로운 통찰력이나 결론을 도출하는 학술적 또는 실무적 글쓰기 형식을 의미합니다.

공간 지능 (Spatial Intelligence)

선행 연구 후보

- AR
- Embodied AI
- Scene-LLM 계열
- PRIMAL 계열
- ImagineNav 계열

연구 분야 (Research Field)

- Embodied AI for XR (확장 현실을 위한 체화된 인공지능)
- Vision-Language-Action (VLA) in Augmented Reality (AR 환경에서의 시각-언어-행동 모델)

핵심 키워드 (Keywords)

- Monocular Scene Understanding (단안 장면 이해)
- Embodied AI Agent (체화된 AI 에이전트)
- Scene-Grounded Motion Synthesis (장면 기반 행동 생성)
- Large Language Models (LLMs)

차별점

- 하드웨어 및 데이터의 대중성 : 오직 일반 스마트폰의 ‘단안(Mono) RGB 영상’ 1개만을 입력 데이터로 사용
- 가상 아바타의 신체적 제약 및 시각적 자연스러움 : 단순 좌표 이동을 넘어, 인간형(Humanoid) 아바타의 관절 움직임(Inverse Kinematics, IK)과 현실 지형의 물리적 제약(바닥 높낮이, 의자 높이 등)을 결합
- 모바일 AR 환경을 위한 실시간성 및 경량화 : 비전 인지 → LLM 판단 → 아바타 물리 제어로 이어지는 복잡한 단계를 모바일 기기에서도 끊김 없이(Real-time) 돌아가도록 최적화하는 파이프라인 경량화 및 지연 시간(Latency) 단축 연구가 핵심 기여도(Contribution)
