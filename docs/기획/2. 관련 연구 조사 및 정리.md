# 세부 분야별 핵심 연구 내용 및 한계점

## 1. 단안 RGB 기반 3D 장면 이해 및 지형 추론

조사해야 할 것

- Monocular depth estimation
- Semantic segmentation
- 3D scene reconstruction (NeRF / SLAM 계열)

핵심 질문

- 단안 RGB로 얼마나 3D 이해가 가능한가?
- 객체 + 공간 구조 + affordance까지 가능한가?

주요 연구 동향

- 단안 카메라(RGB) 한 장 또는 비디오 스트림에서 깊이(Depth)와 3D 바운딩 박스를 추정하는 연구가 주임
    - MonoRec, OmniData, ZoeDepth

기존 연구의 한계점

- 기존 Vision 기반 장면 이해 결과는 객체 수준의 정보에 머물러 있어 행동 가능한 표현으로 직접 활용하기 어려움
- 기존의 객체 탐지(YOLO, Mask R-CNN)나 3D Reconstruction(NeRF, Gaussian)은 공간의 시각적 형태와 라벨만 제공할 뿐, 아바타의 물리적 행동 가능 영역(Navigable area & Actionable geometry)을 직접 도출하지 못함

관련 논문

- Segment Anything Model (SAM)
- ZoeDepth
- Depth Anything V2
- Marigold

## 2. 어포던스(Affordance) 및 공간 지능

조사해야 할 것

- Affordance detection
- Embodied AI benchmark (Habitat, AI2-THOR 등)
- Task-oriented agents

핵심 질문

- AI가 사물을 어떻게 쓸 수 있는지 이해하는가?

주요 연구 동향

- 사물의 시각적 정보로부터 상호작용 가능성을 추론하는 연구(AffordanceNet, VOS)와 3D 공간에서 사람의 포즈를 역추론하는 연구(PSI, PROX)가 있음

기존 연구의 한계점

- 기존 AR 에이전트는 현실 공간을 단순한 장애물로만 인식하여 복잡한 상호작용이 불가능함
- 전통적인 AR 네비게이션은 바닥 평면(Plane Detection)만 인식하여 장애물을 피하는 데 그침
- 사물의 의미론적 기능을 무시하기 때문에 아바타가 오브젝트와 상호작용할 때 물리적 겹침(Clipping)이나 공중 부양 현상이 발생

관련 논문

- AffordanceLLM
- 3D-AffordanceLLM
- AffordanceNet
- PROX
- Marigold

## **3. LLM/VLM 기반 Embodied AI 행동 계획**

조사해야 할 것

- LLM-based planning
- VLA 모델 (RT-2, SayCan 등)
- Scene-LLM 계열

핵심 질문

- LLM이 행동을 잘 결정할 수 있는가?

주요 연구 동향

- 시각 정보와 언어를 결합하여 로봇의 행동을 생성하는 연구(SayCan, VoxPoser, RT-2, Scene-LLM)가 활발함

기존 연구의 한계점

- LLM은 환경의 물리적 제약과 행동 가능성을 충분히 반영하지 못해 비현실적인 행동을 생성함
- LLM은 텍스트 기반으로 행동을 추론하기 때문에 명령을 내릴 때 해당 의자의 실제 3D 높이나 위치, 아바타의 신체적 제약(IK)을 알지 못해 관절이 꺾이거나 현실과 맞지 않는 불가능한 모션을 지시하는 물리적 환각(Physical Hallucination)이 발생

관련 논문

- SayCan
- VoxPoser
- RT-2
- OpenVLA
- SpatialVLM

## **4. AR 환경에서의 캐릭터 제어 및 동작 생성**

조사해야 할 것

- ARKit / ARCore 기반 시스템
- AR avatar interaction 연구
- 게임/산업 사례

핵심 질문

- AR에서 아바타가 현실과 자연스럽게 상호작용하는가?

주요 연구 동향

- 유니티 AR Foundation을 활용한 공간 앵커링 기술과 역운동학(Inverse Kinematics)을 이용해 지형에 발을 맞추는 연구(Foot Planting)가 있음

기존 연구의 한계점

- 모바일 기기의 연산 제약으로 인해 실시간 물리 시뮬레이션을 돌리기 어려우며, 비전 데이터에서 추출한 좌표의 미세한 떨림(Jittering)이 아바타의 모션 불안전성으로 연결됨

관련 논문

- PROX
- PhysDiff

# 주요 선행 연구 정리 및 연구 공백

## 1. 단안 RGB 기반 3D 장면 이해

### Segment Anything Model (SAM) ⭐️⭐️⭐️

연구 목적

- 한 줄 정리 : 어떤 이미지에서든, 어떤 객체에서는, prompt 기반으로 분할할 수 있는 범용 segmentation 모델을 만들자
    - 특정 도메인에 국한되지 않고 어떤 이미지에서든 프롬프트(클릭, 박스, 텍스트)를 통해 사물을 분할할 수 있는 범용 시각 파운데이션 모델 개발
- 핵심 문제
    - 기존 segmentation 모델은 특정 데이터셋에 overfitting되어 새로운 도메인에서 성능 저하
        - segmentation 모델 : 이미지 내의 모든 픽셀이 어떤 물체에 속하는지 하나하나 분류해내는 인공지능 모델
            - 이미지 속의 대상의 정확한 테두리를 따서 배경과 분리해 내는 작업
            - AI가 세상을 단순하게 보는 것을 넘어 사물의 형태와 경계를 완벽히 이해하게 만드는 기술
- 목표
    - Zero-shot으로 모든 객체를 분할할 수 있는 Foundation Model 개발
        - Zero-shot : AI 모델이 학습 과정에서 한 번도 본 적 없는 데이터나 작업을 추가 학습 없이 즉석에서 수행해내는 능력
    - Promptable segmentation task 정의
    - 다양한 prompt(점, 마스크, 텍스트)로 유연한 segmentation

핵심 아이디어

연구 방법

- SA-1B 데이터셋 구축
    - SA-1B : 메타 AI 연구소 FAIR에서 공개한 세계 최대 규모의 이미지 세그멘테이션 (Segmentation) 데이터셋
        - AI가 이미지 속의 물체를 정교하게 구분할 수 있도록 학습시키기 위해 만들어진 거대한 정답지 모음
        - 1100만 장의 고해상도 이미지와 11억 개 이상의 마스크로 구성
            - 마스크 : 물체의 영역을 표시한 데이터
        - 완전 자동화 : 사람이 일일이 영역을 그리는 방식은 시간이 너무 오래 걸리기 때문에 SAM (Segment Anything Model)이라는 모델을 활용해 데이터를 자동 생성했음
    - 1100만 장의 이미지와 10억 개의 마스크로 구성된 SA-1B 데이터셋을 구축하고 모델이 실시간으로 상호작용하며 학습하는 데이터 루프 파이프라인을 설계
        - 즉, 방대한 데이터셋을 구축했다는 것
- 모델 아키텍처
    - SAM = Image Encoder + Prompt Encoder + Mask Decoder
        - Image Encoder : 사진 전체 분석
            - 사진의 전반적인 파악
            - 한 번 실행 후 여러 prompt에서 재사용 가능
        - Prompt Encoder : 원하는 부분 분석
            1. 점 찍기 : 점 주변의 물체 파악
            2. 사각형 그리기 : 영역 안의 물체 파악
            3. 러프한 윤곽선 그리기
        - Mask Docoder : 윤곽선 그리기
            - 결과를 만드는 모듈
- Promptable segmentation
    - 점, 박스, 텍스트 없이도 mask 생성
    - 기존의 segmentation이 class 기반으로 분석했다면, SAM은 점 찍은 부분을 객체로 인식
- 3단계 학습
    1. 사람이 직접 그림 : 전문가가 윤곽선 그림
    2. AI 보조 : AI가 제안 → 사람이 확인
    3. 완전 자동 : AI가 혼자 다 처리

연구 결과

- 제로샷 환경에서도 처음 보는 사물의 경계를 정교하게 분할
- 매우 빠른 속도로 분할
- 모호한 프롬프트에 대해서도 유효한 마스크를 생성

해석

- 시각 인지 분야에서의 GPT 모멘트로 평가받음
    - GPT 모멘트 : 생성형 AI 기술이 대중에게 폭발적인 관심을 받으며 상용화와 확산이 본격화되는 전환점
- 사전 학습 없이도 모든 객체를 인지할 수 있는 능력 입증
    - 학습하지 않은 도메인에서도 작동
- 컴퓨터 비전에서 입력 전처리의 표준 모델이 됨

결론 및 의의

- AR 환경에서 아바타가 상호작용해야 할 대상을 실시간으로 개별 인지할 수 있는 전처리 도구 제공
- 이후 vision taskt에서 객체 분리는 더 이상 연구 대상이 아니게 됨
- 다양한 도메인에서 적용가능한 모델
- 한계점
    - 다만 너무 작은 객체에 대해서는 부정확함
    - 객체의 이름을 모르는 경우 추가 학습 혹은 보조 AI 필요
    - 겹친 객체에 대해서는 한계점이 있음
    - 동영상에서는 안됨

내 연구에서는

- 장면을 객체 단위로 쪼개는 역할
    - 단, 모양만 자르므로 객체의 의미를 모름
    - 따라서 이후 affordance / planning 입력이 필요

### ZeoDepth

연구 목적

- 한 줄 정리 : 단일 이미지에서 실제 거리(미터 단위)를 정확히 예측하자
    - 하나의 모델로 모든 것을 해결하지 말고, 상황별 전문가를 쓰자
- 상대적 거리(Relative Depth)만 추론하던 기존 모델의 한계를 넘어, 실제 미터(Metric) 단위 거리를 정확히 측정
- 단일 RGB 이미지에서 정확한 Metric Depth 추정
    - 사진 한 장에서 거리를 추정
- 핵심 문제
    - 기존 monocular depth는 상대적 깊이만 정확하고 절대 거리는 부정확했음
    - 상대적 깊이 (Relative Depth) : 일반화는 좋지만 미터 단위를 모름
        - 가까운지 먼지만 알려주고 정확한 수치 제공 x
    - 절대 깊이 (Metric Depth) : 특정 데이터 셋에서만 정확
        - 학습한 환경에서만 정확하고 새로운 장소에서는 틀렸음
- 목표
    - 여러 도메인에서 범용적으로 작동
    - Zero-shot generalization + Metric scale 동시 달성

핵심 아이디어

연구 방법

- Relative Depth + Metric Depth를 결합
- 상대적 깊이 추정 모델에 전역적/지역적 미터법 정보를 결합하는 계층적 구조 (Relative-to-Metric) 제안
    - 상대적 깊이를 사전에 학습한 후 → 절대적 깊이를 미세 조정
- Mixture of Experts (MoE) for depth
    - 하나의 모델이 아니라 여러 Metric Depth를 둠
    - 입력 이미지에 따라 적절한 expert를 자동 선택
- 전체 파이프라인
    1. DPT 기반 backbone (Relative Depth)
        - 이미지 → general depth representation 생성
        - 기존 DPT 사용
        - 다양한 데이터에서 학습됐으므로 generalization 확보 가능
        - 여기까지는 상대 깊이 기반
    2. Depth binning + Metric Head
        - 여러 개의 Metric Depth Predictor 존재
        - 각 Predictor는 서로 다른 데이터셋, 스케일, 환경에 특화됨
    3. Depth Binning
        - Depth를 연속값으로 바로 예측하지 않음
        - discrete bin classification + regression 혼합 방식 사용
            - depth range를 여러 구간 (bin)으로 나눔
            - 각 bin에 대한 확률 + refinement
        - ZoeDepth가 다양한 환경에서 동작하는 근거
    4. Router (gating mechanism)
        - 어떤 Mteric Head를 사용할지 결정
        - 이 장면은 어떤 Depth Distribution인가?
    
    정리하면 Image → General Depth Feature → Router → Best Expert → Metric Depth
    
    하나의 백본 (공통 엔진)에서 여러 개의 head를 사용해서 환경 적합성 확보
    
- 다양한 데이터셋을 통한 학습 (Multi-dataset Training)
    - 기존에는 데이터셋마다 Depth Scalse이 다르다는 문제가 있었음
    - 환경마다 expert를 분리하여 학습시키고, router를 통해 선택하는 구조로 보완
        - 따라서 다양한 데이터셋을 통한 학습이 필요
- MBA (Metric Bins Module)
    - 적응형 칸 (Adaptive Bins)을 통해 장면마다 칸 위치 자동 조정
        - 유연한 bin 배치 가능
    - Bin Center와 Widths를 네트워크가 예측

연구 결과

- 절대 거리 정확도 개선
- 다양한 실내외 환경에서 기존 SOTA 모델들을 능가하는 미터법 깊이 정확도 달성
    - 학습하지 않은 데이터셋에서도 잘 동작
- 기존 Metric Depth 대비 평균 28% 향상
- 한계점
    - 투명한 객체에는 한계가 있음
        - 표면을 못 보거나, 경계를 못 찾거나, 반사 때문에 혼란이 있기 때문
    - 너무 어두운 곳에서는 아예 못 봄
    - 카메라 정보가 필요
        - 오래된 사진의 경우 카메라 정보 (EXIF)가 없으므로 정확도 떨어짐
    - 동영상에서 떨림 있음
        - 프레임마다 떨림

해석

- Vsion에서 MoE 구조의 성공 사례
- 상대적 관계 이해가 절대적 거리 측정에 긍정적인 영향을 준다는 것을 입증
- 사용자가 도메인 지정 안 해도 자동 라우팅

결론 및 의의

- 단안 Depth에서 generalization 문제를 구조적으로 해결한 첫 접근
- AR 아바타를 현실 지형 위에 정확한 크기와 거리감으로 배치하는 데 필요한 수치 데이터 제공

내 연구에서는

- 단안 카메라로 실제 거리 추정 가능
- 사물의 정확한 위치 파악 가능
- 아바타 크기 조정

### Depth Anything V2

연구 목적

- 한 줄 정리 : 대규모 데이터 없이도 “어디서든 잘 동작하는” 범용 단안 depth 모델을 만들자
    - 합성 데이터만으로 학습해서 더 빠르고, 더 정밀하고, 더 강력한 거리 측정 AI 개발
- 일반화 성능과 실시간성을 동시에 갖춘 깊이 추정 기초 모델 개발
- depth 분야의 문제점
    - 모델이 특정 데이터셋에 과적합됨
    - 새로운 환경에서 depth 품질 급락
    - 대규모 학습 데이터 부족
- 기존 모델들의 문제점
    - supervised dataset (NYU, KITTI)에 의존
    - 데이터 다양성이 매우 제한적
- 핵심 문제
    - V1의 한계 : 작은 객체 디테일 부족
        - 뭉개져서 보이거나 경계가 흐렸음. 아예 못 보는 경우도 존재
    - Metric Depth 부족
    - 모바일 배포가 어려움
- 목표
    - 더 정밀한 Depth. 특히 더 작은 개체와 얇은 구조에서
    - Metric Depth 추가
    - 모바일 실시간에서 더 빠른 속도

핵심 아이디어

연구 방법

- 라벨 없이도 depth를 학습할 수 있는 foundation model을 만들자
- 가짜 레이블 생성 및 지도학습 기법 개선
    - 가짜 사진 사용 이유 : 유리창, 거울, 전선 등 경계를 확인하기 어려운 객체 문제를 해결하기 위해
        - 컴퓨터가 만든 이미지는 모든 정보가 완벽하게 있기 때문
- 다양한 도메인 학습
- Depth Anything V2의 3가지 축
    1. Self-supervised + Synthentic + Pseudo-label 혼합 학습
    2. 초대규모 데이터 활용
    3. 강력한 ViT 기반 backbone
- 학습 전략
    1. 완벽한 가짜 사진으로 Teacher 학습
    2. Teacher가 진짜 사진에 정답 달기
    3. Student가 Teacher 따라 학습
- 학습 과정
    1. 가짜 사진으로 정확한 분류 학습
    2. 학습한 분류를 실전에 활용
    3. 반복해서 파인튜닝
- 학습 방법
    - 컷믹스 (CutMix)
        - 사진들을 섞어서 다양한 조합 학습
    - 색깔 바꾸기
        - 조명 변화를 통한 다양한 학습
    - 일부 가리기
        - 랜덤하게 일부를 가려서 가려진 부분을 추론

연구 결과

- 대부분 V1과의 비교
- 경계 보존 능력 향상
- 소형 모델로도 초당 수십 프레임의 실시간 추론 성공
- 새로운 환경에서의 정확도 향상
- Teacher의 크기가 클수록 Student도 향상됨
- 한계점
    - 미터 단위는 추가 학습이 필요
    - 가짜와 진짜의 차이가 있음
        - 복잡한 실제 세계에 대한 차이를 이해해야 함
    - 동영상에서는 떨림을 해결하기 위해 프레임을 부드럽게 만드는 처리가 필요함

해석

- 데이터 품질의 정교화가 모델의 크기보다 장면 이해력 향상에 중요함
    - real data보다 인위적으로 만든 synthetic data가 학습에 유의미한 영향을 끼침
    - 기존에는 진짜 데이터로 학습해야 진짜 같다는 생각이 있었는데 가짜 데이터가 더 효과가 좋다는 결과를 냄
        - 가짜 데이터는 완벽한 정답을 제공하고 진짜 데이터는 틀린 정답을 제공하는 차이점이 있었음
        - 가짜 데이터라도 완벽한 정답이 중요하다는 시사점

결론 및 의의

- 성능 제약이 있는 모바일 기기에서 지형지물 배경을 실시간으로 파악하기 위한 실무적인 연구
- 소형 모델로 속도/정확도의 트레이드 오프 해결
- 가짜 데이터를 이용한 AI 학습 방식 제안
- 스마트폰, 노트북, 고성능 PC 등 모든 기기에서 사용할 수 있음

내 연구에서는

- ZeoDepth와 Depth Anything V2 중 선택 필요
    - ZeoDepth
        - 경계가 다소 흐림
        - 작은 물건은 놓침
    - Depth Anything V2
        - 경계가 명확
        - 작은 물건도 잘 봄
        - 스마트폰에서도 사용 가능
- 스마트폰 카메라로 비디오 촬영 → Depth Anything V2로 분석 → SAM으로 물체 탐색 → 아바타 배치

### Marigold

연구 목적

- 한 줄 정리 : 확률적 생성 모델 (Diffusion)을 이용해서 더 정밀하고 안정적인 depth를 추정하자
- 기존 단안 깊이 추정 방법의 문제점
    - 학습 데이터에 포함되지 않은 새로운 장면이나 레이아웃에서 성능이 크게 저하됐음
    - deterministic 모델은 하나의 정답만 예측
    - 복잡한 장면에서 depth 붕괴
    - 특히 반사, 투명, 얇은 구조에서 depth가 쉽게 틀림
- Diffusion 모델을 이용한 Depth 추정
    - Stable Diffusion은 인터넷의 수십억 이미지를 보고 세상이 어떻게 생겼는지 학습했으므로, 이 지식을 깊이 추정에 활용해보자
- 사전 학습된 Diffusion 이미지 생성 모델에서 풍부한 시각 지식을 활용하여 고품질의 깊이 맵 생성
    - Diffusion 모델 : 멀쩡한 이미지에 노이즈를 섞어 망가뜨린 뒤, 이를 다시 복구하는 모델
    - 가설 : 단안 깊이 추정의 본질은 포괄적인 시각적 세계 이해이며, 이는 인터넷 규모 데이터로 학습된 생성 모델에 이미 내재되어 있다

연구 아이디어

연구 방법

- Stable Diffusion 기반으로 depth 생성
    - depth를 예측하는 것이 아닌 생성
    - Stable Diffusion으로 이미지를 생성하는 것이 아닌 깊이 추정에 사용
        - RGB 이미지로 → Depth map 생성
    - VAE는 그대로 사용
    - U-Net 약간 수정
    - Text Encoder 사용 x
- Diffusion의 반복 정제 과정을 이미지 생성 과정을 깊이 맵 생성 과정으로 용도 변경하여 파인 튜닝
    - Diffusion이 점진적으로 노이즈를 걷어내는 것처럼 depth를 한 번에 예측하지 않고 여러 step을 거쳐 점진적으로 생성
    - 일반 노이즈 대신 다중 해상도 노이즈 사용
        - 큰 패턴 노이즈 + 중간 패턴 + 작은 디테일 노이즈 혼합
        - 처음에는 거친 노이즈로 시작해서 → 점점 섬세하게
            - 조각에서 대리석을 뭉텅이로 깎다가 디테일하게 다듬는 것처럼
- RGB 이미지를 조건 (condition)으로 사용
    - RGB 구조를 참고하면서 depth를 생성
- 품질 검사 (Test-time Ensembling)
    - 같은 이미지를 10번 다르게 예측하고 중앙값 선택
    - Stable Diffusion이 학습한 지식을 깊이 추정에 활용해서 처음 보는 장면을 이해하는 것
- Image → Latent → Depth 순서로 생성

연구 결과

- 사진 한 장에서 노이즈가 극히 적고 미세한 굴곡까지 살아있는 정교한 깊이 정보 추출
- detail 복원이 뛰어나짐
    - 경계와 얇은 구조, 복잡한 geometry에서
- 합성 데이터 학습으로 인한 성능 개선
    - 실제 센서 데이터는 거리와 반사 표면, 가려진 곳으로 인해 측정 실패가 있었음
    - 합성 렌더링을 통해 모든 픽셀에 대해 정확한 깊이 값을 사용
- 여러 번의 확률적 예측을 한 후 종합해서 결과 산출
- 한계점
    - 속도가 느림
    - 확률적인 특성이 강해 일관성 문제가 있음
    - 합성 데이터 품질에 민감
    - 계산 비용이 많이 필요함

해석

- 단안 깊이 추정의 본질은 포괄적인 시각적 세계 이해라는 가설을 실험적으로 입증했음
    - 기존에는 더 많은 depth 데이터 수집으로 학습하여 데이터 다양성 한계에 있었음
    - Marigold는 이미지 생성 모델의 prior을 사용하여 일반화에 활용
    - 데이터 효율성 향상
- Depth를 생성 문제로 본 접근
- 생성형 AI의 잠재적 시각 공간 이해도가 단순 판별 모델보다 뛰어날 수 있음을 입증
- 합성 데이터의 우월성
    - 실제 센서 데이터 대신 합성 데이터만 사용할 경우가 더 좋다

결론 및 의의

- 정밀한 표면 데이터를 제공하여 시각적 오류를 최소화함

내 연구에서는

- 단안 RGB 기반 3D 장면 이해에 활용 가능

## 2. 어포던스(Affordance) 및 공간 지능

### AffordanceLLM ⭐️⭐️⭐️

연구 목적

- 한 줄 정리 : LLM을 활용해 시각 정보로부터 행동 가능성 (Affordance)를 추론하자
    - VLM에 내재된 세계 지식과 인간-물체 상호작용 지식을 활용하여 학습 시 보지 못한 물체에 대해서도 affordance를 정확히 예측하자
- 2D 이미 상에서 사물의 기능 (Affordance)을 LLM의 지식으로 추론하고 해당 부위를 좌표와 연결
- 기존 비전 모델의 한계
    - 물체를 인식하고 위치를 파악할 수 있지만 이것으로 무엇을 할 수 있는가는 몰랐음
- Affordance Grounding의 난이도
    - Affordance Grounding : 이미지의 특정 행동(action)을 수행할 수 있는 물체의 영역을 찾아내는 작업
    - 필요한 작업
        - 물체 및 부품의 검출, 위치 파악, 인식
        - 장면의 기하학적 배치/레이아웃
        - 3D 형상 및 물리
        - 물체의 가능성과 잠재적 상호작용
        - 인간-물체 상호작용 패턴
    - 위의 작업의 경우 대부분 한정된 학습 데이터의 라벨링한 정보만으로는 얻을 수 없었음
    - 이에 LLM, VLM이 학습한 일반 지식을 이용

연구 아이디어

- 대규모 VLM의 세계 지식을 어포던스 그라운딩으로 전이(transfer)
    - Affordance를 학습시키는 것이 아니라 LLM이 추론하게 하자
    - 사물의 용도는 인간의 언어 지식 속에 정의되어 있으므로 VLM을 통해 언어적 개념과 이미지의 특성을 공간적으로 일치시킬 수 있다
    - 외관(appearance)은 다양하지만, 기하학(geometry)은 본질적이기 때문
- 최소 변경, 최대 활용
    - 고정 (Frozen)
        - VLM의 언어 지식
        - 시각 인코더
    - 새로 학습
        - visual → text embedding
        - Mask decoder
- 작동 원리
    1. LLM이 어포던스를 언어적으로 이해
    2. mask token이 그 이해를 암호화
    3. decoder가 spatial map으로 변환
- 입력값은 RGB + Depth

연구 방법

- 파이프라인
    - Image → Object Recognition → Text prompt → LLM → Affordace Reasoning
    1. 이미지 입력
    2. 객체 탐지/인식
    3. 객체를 텍스트로 변환
    4. LLM에 입력
    5. affordance 출력
- 모델 구조보다 프롬프트 설계가 핵심
    - LLM이 이미 물리적 상식과 인간 행동 지식을 가지고 있기 때문
    - 즉, Affordance는 학습이 아니라 상식 기반 추론
- 멀티 모달 LLM(VLM)을 활용해 “어디를 조작해야 하는가?”라는 질문에 답하게 하고, 응답 텍스트와 픽셀 좌표(Grounding)를 연결하여 학습

연구 결과

- 사전 정의된 라벨 없이도 자연어 명령에 대해 연관된 사물의 위치와 면을 정확히 가리킴
- unseen object에서도 잘 동작
- zero shot 가능
    - 데이터셋이 필요 없음
    - 새로운 객체 대응 가능
- 한계점
    - 물리적 정확도 부족
        - 실제로 가능한 행동이 아닐 수도 있음
    - hallucination
        - 존재하지 않는 affordance 생성
    - 공간 정보 부족
        - 위치와 거리를 고려 안 하는 경우
    - 너무 느린 속도

해석

- 인공지능이 사물의 기하학적 형태를 넘어 ‘인간 중심의 도구적 가치’를 이해하기 시작
- affordance = 지식 + 추론 문제로 정의
- 핵심 통찰의 이론적 함의

결론 및 의의

- AR 아바타가 사용자의 의도에 맞춰 현실 사물의 특정 부위와 상호작용하게 만드는 논리적 교량
- 행동 가능성 추론을 데이터가 아닌 LLM으로 해결한 첫 접근
    - affordance를 위한 데이터 의존 탈피
- VLM의 새로운 활용 패러다임 제시
- 어포던스 그라운딩에서 3D 정보의 중요성 입증

내 연구에서는

- 사물 어포던스 및 기능적 공간 추론
- 아바타와 사물 간의 인터랙션

### 3D-AffordanceLLM ⭐️⭐️⭐️

연구 목적

- 한 줄 정리 : LLM 기반 affordance 추론을 3D 공간으로 확장하여 “행동 가능한 위치와 방식”까지 도출하자
    - 3D 공간 데이터에서 언어적 추론을 수행하여 아바타가 앉거나 잡아야 할 정확한 xzy 좌표와 법선을 추출하는 공간 지능 모델
- 2D를 넘어 3D 클라우드 상에서 사물의 기능과 정밀한 3D 상호작용 좌표, 그리고 자세 방향성을 추론
- 기존 3D 어포던스 검출 방법의 한계
    - 한정된 라벨과 단순 텍스트 매칭만으로는 유연하고 맥락을 이해하는 어포던스 추론이 불가능
    - 정해진 명령만 이해
    - 단순 텍스트만 처리
        - 복잡한 맥락 이해 불가
    - 고정된 어포던스 라벨 세트에 의존
        - 미리 정의된 라벨만 인식
    - 제한된 일반화
        - 학습 데이터에 없는 물체에는 취약함
- Affordance 구조
    1. Semantic affordance : 무엇을 할 수 있는가
    2. Spatial affordance : 어디서/어떻게 할 수 있는가
- AffordanceLLM은 Semantic affordance만 해결했음

연구 아이디어

- LLM + 3D Scence Representation을 결합해서 행동 가능한 공간을 생성한다
- 아바타의 실제 행동을 위해서는 2D 좌표만으로는 부족하며, 사물의 입체적인 면의 높이와 기울기를 알아야 한다
    - 이를 위해 3D 기하 데이터를 직접 LLM 입력 토큰으로 주입
- LLM이 가지고 있는 기본 상식을 3D 물체 이해해 활용하자
    - 사용자 질문 → GPT 이해 → AFF 토큰 반환 → 3D 마스크로 변환
- 핵심 구조
    - RGB/Point Cloud → 3D Scence Representation → Object + Geometry Extraction → LLM Reasoning → Affordance + Spatial Region

연구 방법

- 3D 포인트 클라우드를 인코딩하여 토큰화한 뒤 LLM에 입력
- 상호작용 지점의 위치뿐 아니라 아바타가 취해야 할 자세의 방향성까지 함께 예측하도록 대규모 3D 어포던스 데이터셋을 학습시킴
- 전체 파이프라인
    
    3D Scence → Scene Parsing → Language Interface 생성 → LLM 기반 Affordance 추론 → Spatial Grounding
    
    1. 3D Scence
        - 입력으로 RGB-D, Point Cloud, NeRF / 3D reconstruction을 받음
        - 즉, 공간이 있는 입력
    2. Scene Parsing
        - 공간을 객체, 위치, 기하 구조로 분해
    3. Language Interface 생성
        - 3D를 적절한 텍스트로 변환하여 LLM에 입력
        - 즉, 3D → Language abstraction
    4. LLM 기반 Affordance 추론
        - LLM이 행동 종류 추론, 조건 판단, 공간 조건 결합
    5. Spatial Grounding
        - LLM 출력을 3D 좌표로 매핑
- Language as Interface
    - 3D 문제를 언어 문제로 바꾸는 것이 가장 중요
    - geometry → text → reasoning → geometry 순으로 변환
- Compositional Reasoning
    - LLM이 단순 매핑이 아니라 물리적 조건, 기능적 조건, 상황 조건을 동시에 고려

연구결과

- 아바타와 사물의 정확한 3차원 각도를 계산
- 한계점
    - 3D 정보를 text로 변환하면서 정보 손실
    - 애매한 질문 해결 불가
    - 너무 작은 객체는 인식 불가
    - 속도가 너무 느림
    - 색상 정보가 없음

해석

- LLM을 3D reasoning 엔진으로 확장
    - perception → language → reasoning → action의 순서
- LLM의 추론력이 실제 물리적 3차원 좌표계와 완전히 통합되어 공간적 상식을 발휘할 수 있음을 입증
- 자연어 추론을 통한 3D 어포던스 탐색이 고정 라벨 기반보다 훨씬 효과적이다

결론 및 의의

- 로봇이나 AR 아바타가 물리적 세계를 이해하고 행동하는 새로운 패러다임 3D-Reasoning 제시
- LLM과 3D 어포던스의 통합
- 자연스러운 대화를 통한 어포던스 추론

내 연구에서는

- 행동의 좌표와 의미가 동시에 생성됨

## **3. LLM/VLM 기반 Embodied AI 행동 계획**

### SayCan ⭐️⭐️⭐️

연구 목적

- 한 줄 정리 : LLM의 계획 능력과 로봇의 실제 수행 가능성을 결합하여 “실행 가능한 행동 계획”을 생성하자
    - 거대 언어 모델의 고차원적 논리 추론 능력 + 로봇의 하위 실행 능력을 확률적으로 결합
        
        → 현실의 물리적 제약을 반영한 장기 지능적 행동 계획 수립
        
- LLM에게 현실적인 몸을 달아준 연구
- 기존 문제점
    - LLM은 해야 할 일은 잘 계획하고, 로봇은 할 수 있는 일만 수행 가능
    - 하지만 LLM이 말한 행동이 로봇이 실제로 가능한 행동이 아니었음
    - Physical Grounding의 부재
        - LLM은 상황에 대한 해답은 알고 있지만 물리적 한계는 인지하지 못했음

핵심 아이디어

- 의미적으로 맞고 + 물리적으로 가능한 행동만 선택
- 해야 하는 것(Say)과 할 수 있는 것(Can)을 곱해서 행동을 선택
- 확률적 결합
    - 행동 선택의 기준을 ‘언어적 타당성’과 ‘물리적 가능성’의 곱으로 정의
- Affordance 도입
    - 로봇의 각 스킬이 현재 상태에서 성공할 확률을 Affordance로 정의하고, 이를 LLM의 시맨틱 점수와 실시간으로 결합함

연구 방법

- LLM을 Planner로 사용
    - rule-based에서 벗어남
- Grounding
    - 추상적인 LLM의 응답을 현실에 결합
- 시스템 아치택처
    - LLM (Scoring) : 명령 i가 주어지면, 가능한 행동 후보 @에 대해 텍스트 기반 로그 확률인 P(@|i) 산출
    - Affordance (Value Function) :
- 전체 파이프라인
    
    Instruction (자연어) → LLM 행동 후보 생성 → 각 후보에 대해 LLM score (semantic)과 Robot skill score (affordance) 생성 → Score 결합 → 최적 행동 선택 → 실행 → 반복
    
    1. 행동 집합 정의 (Skill Library)
        - 로봇이 할 수 있는 행동 정의
    2. LLM 기반 행동 확률 계산
        - 목표에 적합한 행동 계산
    3. Affordance Score 계산 (Robot side)
        - 각 행동에 대해 로봇이 현재 상태에서 실행 가능한지 평가
    4. Score 결합
        - semantic x affordance
        - 최종적으로 높은 Score의 행동 선택
    5. Iterative Planning
        - 행동 선택 → 실행 → 상태 업데이트 → 다시 선택을 반복

연구 결과

- LLM 단독보다 안정적
- 복잡한 task 수행
- zero-shot instruction 대응 가능
- 유연성
    - 차선책을 선택하는 능력을 보임
- 한계점
    - Skill Library 의존
        - 새로운 행동 불가능
    - vision → LLM 연결이 약함

해석

- LLM을 현실 세계에 연결하는 첫 구조
    - LLM이 현실을 잘 모르므로 현실 판단을 로봇에게 맡김으로써 해결
    - 역할 분리
        - LLM은 계획
        - 로봇은 실행 가능성 판단
    - VoxPoser, RT-2, OpenVLA로 확장됨
- Semantic Grounding
    - 언어 모델이 현실 세계의 물리적 자아(Body)를 가졌을 때 어떻게 상식을 행동으로 번역하는지를 보여줌
- LLM을 새로 학습시키지 않고, 기종 모델을 고정(Frozen)시킨 상태에서 로봇의 가치 함수만 인터페이스로 연결하여 강력한 성능을 냈음

결론 및 의의

- LLM의 추론과 물리적 실행 가능성을 결합한 최초의 실용적 프레임워크
    - LLM의 지시와 로봇의 능력을 확률적으로 결합
    - embodied AI 시작점
    - 현실 적용 가능성 확보

내 연구에서는

- 전체 시스템의 의사결정 구조
- 물리적 정합성 강화
    - 물리적 가능성과 동떨어진 LLM의 추천은 필터링을 통해 배제

### VoxPoser ⭐️⭐️⭐️

연구 목적 

- 한 줄 정리 : 자연어 명령을 3D 공간상의 행동 가능한 위치(affordance)로 직접 변환하자
    - 언어와 공간의 연결
    - VoxPoser는 자연어 명령을 3D 공간상의 value map으로 변환하여 최적의 행동 위치를 계산하는 모델
    - LLM이 작성한 코드를 통해 3D 복셀 (Voxel) 공간에 가치 지도를 생성하고, 이를 기반으로 물리적으로 타당한 경로를 도출
- LLM이 직접 관절 각도를 계산하는 대신, 3D 공간 상에 가치 지도를 그려 동작을 가이드하는 유연한 행동 생성 프레임워크 제안
- 행동을 좌표가 아니라 3D value map으로 표현
    - 기존 방식이 특정 좌표를 선택했다면, VoxPoser는 공간 전체에 대해 행동 가능성 점수를 부여
- 기존 시스템은 대부분 motion primitive를 수동으로 설계하고, 각 primitive를 학습시키기 위한 대규모 로봇 데이터가 필요했음
    - 또한 세밀한 제어가 어려웠음

연구 아이디어

- LLM은 공간적 관계를 글로 생성하는 데 능하므로, LLM에게 ‘공간의 각 위치에 점수를 매기는 코드’를 짜게 하여 물리 엔진이 이를 최적 경로로 해석
- LLM을 통해 명령을 3D value map으로 변환하고, 이를 trajectory optimiaation의 objective function으로 사용
    - Value Map : 3D voxel 공간에서 각 위치에 가치를 할당한 map
    - Value : High value는 끌림 (attraction)을, Low value는 밀침 (repulsion), Zero는 중립(neutral)을 나타냄

연구 방법

- LLM이 파이썬 코드를 생성하여 3D 복셀 그리드에 가치 정보를 할당
    - 목표물에는 끌개 (Attractor), 장애물에는 밀개 (Repulsor) 점수를 부여하는 가치 지도 (Value Map)를 생성하고, 이를 기반으로 최적의 궤적 (Trajectory)을 산출
- 전체 파이프라인
    
    Language Instruction → LLM Decomposition → Affordance Function Generation → 3D Value Map 생성 → 최적 위치 선택 → Robot Action 실행
    
    1. Language Instruction
        - 명령어 입력
    2. LLM Task Decomposition
        - LLM의 명령어 쪼개기
        - 명령어에 따른 정보 분해
        - 행동에 따른 task 분해
    3. Affordance Function
        - 함수 형태의 행동 기준 생성
    4. 3D Value Map 생성
        - 환경을 voxel grid로 변환
        - 각 voxel에 대해 함수 계산
        - 공간 전체가 행동 지도가 됨
    5. Optimization
        - 가장 높은 score를 가진 위치 선택
    6. Motion Planning
        - 정해진 위치로 로봇 이동
        - trajectory 생성
- Language → Function 변환
    - 기존 LLM이 텍스트만 출력했다면 VoxPoser는 행동 함수 생성
    - LLM으로 프로그램을 만드는 것

연구 결과

- 사전 학습되지 않은 복잡한 동작도 장애물을 피해 유연하게 수행
- 복잡한 조작 작업 가능
- 다양한 환경에 강점
- zero-shot instruction 가능
- rule-based 제거
- 자연어 인터페이스 가능
- 한계점
    - 물리 정확도 문제
    - LLM 의존성으로 인한 잘못된 함수 생성
    - 계산 비용이 너무 큼

해석

- LLM을 물리 행동 생성기로 확장한 첫 구조
    - language → function → space → action
    - 행동을 계산
- 언어 모델의 지능을 물리적 좌표계와 연결하는 가장 직관적이고 강력한 인터페이스임을 증명함
- 기존 LLM for robotics가 primitive에 종속적인 것을 탈피함

결론 및 의의

- 자연어 기반 행동을 3D 공간의 최적화 문제로 변환한 최초의 접근
    - LLM → 실제 행동 연결
    - embodied AI 핵심 기술
    - 로봇, AR, XR 모두 적용 가능
- LLM의 추론력을 하부 모션 제어기와 분리하면서도 긴밀하게 연결하는 새로운 아키텍처 제시

내 연구에서는

- 행동 자동 생성
    - 자연어와 상황을 보고 행동 함수 생성
    - 공간 전체 평가
    - 최적 행동 선택
- 아바타가 현실 지형을 인지하고 벽이나 가구에 부딪히지 않도록 실시간 NavMesh 및 이동 경로를 생성

## 번외 연구

### LEO

연구 목적

- 한 줄 정리 : 언어를 중심으로 perception → reasoning → action을 하나의 모델로 통합한 generalist
    - 3D 세계 내에서 시각적 지각, 상식적 추론, 정교한 동작 제어를 단일 신경망 구조로 통합하여 수행하는 Embodied Generalist Agent 구축
- Embodied AI를 language foundation model로 통합한 최초의 generalist 접근
    - AI에게 3D 눈과 물리적 신체를 동시에 부여
    - 개별 모듈로 분절되어 있던 인지, 판단, 행동을 하나의 거대한 모델로 통합한 ‘Embodied Foundation Model’을 제시
- 기존 embodied AI의 문제점
    - perception, reasoning, action이 완전히 분리되어 있음
    - 모듈 간 정보가 손실
    - 확장성 부족
    - 새로운 task 대응 어려움

핵심 아이디어

- 모든 것을 Language Latent Space로 변환해서 하나의 모델로 처리
    - perception, action도 언어처럼 다룸
- 통합 토큰화
    - 3D 공간 정보, 언어 명령, 로봇의 행동 수치를 모두 동일한 형태의 토큰으로 변환하여 대규모 트랜스포머 모델에 입력
- LLM의 3D화
    - 언어만 학습한 LLM에 3D 기하학적 인지 능력 주입

연구 방법

- Language as Universal Interface
    - 언어는 모든 것을 표현할 수 있는 공통 인터페이스다라는 철학
- 시스템 아키텍처
    1. Multi-modal Encoder
        - 이미지, 3D scene, 텍스트 instruction을 입력받아 모두 하나의 embedding space로 변환
    2. Language-Centric Transfomer
        - 모든 정보를 language token처럼 처리
    3. Action Decoder
        - 텍스트, 행동으로 출력

연구 결과

- multi-task 수행
- zero-shot generalization
- 하나의 모델로 질문 답변, 행동 생성, 환경 이해 모두 수행
- 한계점
    - 매우 큰 데이터가 필요함
    - 실제 물리 정확도가 부족함
    - 해석 가능성이 낮음
    - 계산 비용이 큼

해석

- Embodied AI를 LLM 패러다임으로 재정의해 연구
- 기존에는 pipeline 기반 시스템이었다면 LEO는 foundation model 기반 연구
- 2D 이미지를 3D로 해석하는 과정을 생략하고 직접 3D 데이터를 처리함으로써 공간적 정확도를 향상시킴

결론 및 의의

- 인간처럼 보고, 생각하고, 움직이는 능력을 하나의 모델에 담아냄
- Embodied AI 분야에서 거대 기초 모델(Foundation Model)의 가능성을 3D 환경으로 확장

내 연구에서는

- pipeline이 아닌 모든 것을 하나로 처리 가능
    - 구조 단순화
    - 확장성 향상

연구 공백

1. 통합 부재: Vision-LLM-Motion이 따로 연구됨
2. 모바일 제약: 대부분 고성능 환경 가정
3. 정적 이해: 동적 affordance 추론 부족
4. 물리적 환각: LLM이 불가능한 행동 지시

# 본 연구의 핵심 차별점

본 연구는 단안 RGB 기반 장면 이해 결과를 affordance-aware representation으로 변환하고, 이를 LLM 기반 행동 결정과 결합하여, AR 환경에서 자연스러운 아바타 행동을 생성하는 통합 파이프라인을 제안한다

기존의 Embodied AI 연구들이 주로 고가의 센서(RGB-D, LiDAR)와 강력한 GPU 서버를 기반으로 하나 로봇 제한에 초점을 맞췄다면, 본 연구는 대중적인 모바일 기기(단안 RGB)만을 활용하여 실시간 3D 장면 이해와 LLM의 행동 판단을 AR 환경에 맞게 경량화하고 통합했다는 점에서 독창성을 가짐

Vision의 의미론적 분석을 행동 후보군(Action Candidates)로 변환하여 LLM에 제공함으로써 물리적 환각을 최소화하는 Scene-to-Affordance 구조를 가짐
