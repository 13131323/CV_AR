실시간 AR 파이프라인에서 비전 모듈의 오탐지를 보정하고 어포던스를 추출하기 위해 Gemini(VLM)를 연동하는 과정에서 아키텍처 설계가 변경되었다. 

초기에는 비전 파이프라인에서 추출한 기하학적 수치 데이터만을 JSON 형태로 LLM에 전송하여 논리적으로 오탐지를 보정하려고 했으나, 수치 데이터 자체가 부정확할 경우 한계가 있어 원본 카메라 이미지를 함께 전송하는 멀티모달 방식으로 전환하였다. 

하지만 하나의 프롬프트에서 객체 오인식 보정과 어포던스 추출을 동시에 지시할 경우 작업 집중력이 희석되어 퀄리티가 떨어질 우려가 제기되었다. 이를 해결하기 위해 두 작업을 별개의 API 호출로 분리하는 방안을 고려했다. 그러나 어포던스라는 것은 사물이 놓인 주변 배경 환경에 대한 시각적 맥락이 반드시 필요하기 때문에, 만약 어포던스 추출 단계를 분리해서 텍스트만 주게 되면 배경 맥락을 잃게 되고 반대로 이미지를 두 번 주게 되면 실시간 AR 환경에서 지연 시간과 비용이 두 배로 증가하는 딜레마가 발생했다. 

이러한 문제를 모두 해결하기 위해 최종적으로 단일 VLM 호출에 구조화된 Chain-of-Thought(CoT)를 적용하는 아키텍처를 채택했다. LLM이 한 번에 결론을 내놓아 집중력이 분산되는 것을 막고자, JSON 출력 스키마의 순서를 엄격하게 강제했다. LLM은 먼저 이미지의 주변 배경과 환경을 서술하고, 그 다음 객체의 진짜 정체성을 보정하고 물리적 상태를 판단한다. 그 후 앞서 파악한 배경과 상태를 종합하여 아바타가 할 수 있는 행동을 논리적으로 전개한 뒤 최종 어포던스 태그를 부여하게 된다. 

더불어 API 호출 비용과 지연을 최소화하기 위해 매 프레임마다 이미지를 전송하는 대신 화면 내 객체의 개수나 면적, 깊이 등의 기하학적 수치가 크게 변했을 때만 이미지를 전송하여 결과를 갱신하는 동적 전송 로직도 함께 적용하기로 했다. 이 아키텍처를 통해 빠른 실시간성을 유지하면서도 배경을 고려한 정확한 객체 인지와 어포던스 판단이 모두 가능해질 것이다.
