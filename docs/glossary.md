# 용어 정리 (Glossary)

> 3D 좌표 파트 및 인터페이스 문서에서 쓰는 용어를 정리. VLM 파트·신규 팀원 참고용.

## 선행 모듈 / 모델

| 용어 | 뜻 |
|---|---|
| **단안(Monocular) RGB** | 카메라 1대(일반 웹캠)만 사용. 스테레오/깊이센서 없이 컬러 영상만 입력. |
| **YOLO (YOLOv8)** | 실시간 객체 검출기. 이미지에서 객체의 **클래스**와 **2D 박스(bbox)** 를 찾음. |
| **SAM (Segment Anything)** | 객체를 픽셀 단위로 오려내는 **분할(segmentation)** 모델. 박스를 주면 그 객체의 **마스크**(픽셀 영역)를 반환. |
| **Depth Anything V2 (Metric)** | 단안 이미지에서 픽셀별 **깊이(거리)** 를 추정하는 모델. "Metric"판은 결과가 **미터 단위**. |
| **Metric Depth Map** | 위 모델이 낸, 각 픽셀의 거리(m)를 담은 2D 배열. |

## 카메라 / 기하

| 용어 | 뜻 |
|---|---|
| **카메라 내부 파라미터 (Intrinsics)** | 렌즈 고유값 `fx, fy`(초점거리), `cx, cy`(주점). 픽셀↔실제 각도 변환에 필요. |
| **Camera Calibration** | 체커보드 촬영으로 위 내부 파라미터를 실측하는 과정. 기기마다 다름. |
| **광학 중심 (Optical Center)** | 렌즈를 지난 빛이 모이는 점. 우리 좌표계의 **원점(0,0,0)**. 쉽게 "렌즈 위치". |
| **주점 (Principal Point, cx·cy)** | 광축이 이미지에 닿는 픽셀 좌표. 대략 화면 중앙. |
| **핀홀(Pinhole) 역투영** | 픽셀(u,v)+깊이(Z)를 실제 3D 좌표(X,Y,Z)로 되돌리는 계산. `X=(u−cx)·Z/fx`. |
| **좌표계 `camera_opencv_meters`** | 원점=광학중심, 단위=m, 축 `+X`오른쪽/`+Y`아래/`+Z`정면(OpenCV 오른손). |
| **카메라 상대 좌표** | 값이 "그 프레임의 카메라 기준"이라는 뜻. 방 기준 절대좌표(월드좌표) 아님. |

## 대표값 산출 / 통계

| 용어 | 뜻 |
|---|---|
| **무게중심 (Centroid)** | 마스크에 속한 모든 픽셀 좌표의 평균. 객체의 대표 픽셀(u,v). |
| **대표 깊이 (Representative depth)** | 마스크 영역 깊이값들을 대표하는 하나의 Z값. 우리는 median 사용. |
| **중앙값 (Median)** | 값들을 정렬했을 때 가운데 값. 이상치에 강함(평균과 대조). |
| **이상치 (Outlier)** | 대다수와 동떨어진 값. 예: 마스크 경계가 문 배경의 먼 깊이. |
| **IQR / Tukey 1.5×IQR 펜스** | 사분위 범위(Q3−Q1) 기반 이상치 경계. `[Q1−1.5·IQR, Q3+1.5·IQR]` 밖을 제거. 임계값이 데이터 분포에서 자동 결정됨. |
| **inlier_ratio** | 이상치 제거 후 남은 픽셀 비율. 1에 가까울수록 깨끗. |

## 안정화 (프레임 간)

| 용어 | 뜻 |
|---|---|
| **지터 (Jitter)** | 실제로 안 움직이는데 값이 프레임마다 미세하게 떨리는 현상. |
| **좌표 안정화 (Stabilization)** | 지터를 눌러 좌표를 매끄럽게 만드는 처리. |
| **One Euro Filter (1€ 필터)** | 속도 적응형 저역통과 필터. 느릴 땐 강하게(지터↓), 빠를 땐 약하게(지연↓) 걸림. |
| **EMA** | 지수이동평균. 단순 스무딩. 고정 계수라 지터/반응성 트레이드오프 고정(그래서 1€ 채택). |
| **IoU 매칭** | 프레임 간 같은 객체 연결에 쓰는 박스 겹침 비율(교집합/합집합). |

## 인터페이스 / 어포던스

| 용어 | 뜻 |
|---|---|
| **Object Representation** | 객체 1개를 표현하는 구조화 데이터(식별정보+3D좌표+공간정보). VLM 입력의 단위. |
| **VLM** | Vision-Language Model. 이미지+텍스트로 추론하는 모델(여기선 정체성·어포던스 판단). |
| **어포던스 (Affordance)** | 객체로 **할 수 있는 행동**(예: 쥐기, 누르기, 앉기). VLM이 **출력**으로 결정. |
| **PPS (Peripersonal Space)** | 손 닿는 근접 공간. 본 프로젝트 임계 **0.7m**. 상호작용 가능 판정에 사용. |
| **bbox_2d** | 2D 바운딩 박스 `[x1,y1,x2,y2]`(픽셀). |
| **centroid_2d / centroid_y** | 마스크 중심 픽셀 좌표 / 그 y성분. |
| **spatial_3d** | 객체의 3D 좌표 묶음(x,y,z,distance_from_agent 등). |
| **distance_from_agent** | 카메라(=아바타)로부터 객체까지 유클리드 거리(m). |

## 깊이 보정 / 검증 지표

| 용어 | 뜻 |
|---|---|
| **pred (원시 예측)** | depth 모델이 낸 보정 전 거리값. |
| **scale · offset** | 선형 보정 계수. `true = scale·pred + offset`. 기기별로 다름. |
| **provenance** | 계수의 출처·이력(어떤 측정으로 얻었는지). `depth_scale.json`에 기록. |
| **포화 (Saturation)** | 실제 거리가 늘어도 모델 pred가 더는 안 커지는 현상. 원거리에서 발생. |
| **AbsRel** | 평균 절대 상대오차 = mean(|pred−true|/true). 낮을수록 정확. |
| **δ₁ (delta-1)** | max(pred/true, true/pred) < 1.25 인 표본 비율. 단안 깊이 표준 정확도 지표. 1.0이 최고. |
| **RMSE / MAE** | 제곱근평균제곱오차 / 평균절대오차. 둘 다 작을수록 정확(단위 m·cm). |
| **R²** | 회귀 적합도. 1에 가까울수록 직선이 데이터를 잘 설명. |
